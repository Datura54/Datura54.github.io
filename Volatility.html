Memory Dump Analysis and Malware Detection using Volatility

Introduction

In this project, the focus is on analyzing a memory dump of a compromised system using the Volatility tool. The following steps are involved:

Extract Volatility: Change the directory to "/mnt/c/IntroLabs/" and extract the Volatility files using the command `tar xvfz ./volatility3-1.0.0.tar.gz`. Navigate to the "volatility3-1.0.0/" directory.

Analyze network connections: Run the command `python3 vol.py -f /mnt/c/tools/volatility_2.6_win64_standalone/memdump.vmem windows.netscan` to examine the network connections in the memory dump. Look for any suspicious connections, such as SMB (port 445) connections to internal systems.

Investigate processes: Use the command `python3 vol.py -f /mnt/c/tools/volatility_2.6_win64_standalone/memdump.vmem windows.pslist` to view the list of processes in the memory dump. Look for any unusual or suspicious processes, such as cmd.exe, which is not commonly seen in normal user interactions.

---snap---

Trace parent processes: Use the command `python3 vol.py -f /mnt/c/tools/volatility_2.6_win64_standalone/memdump.vmem windows.pstree` to trace the parent processes of suspicious processes. This can help create a timeline of actions on the system. Note the parent process of TrustMe.exe and the invocation by Explorer.exe.

Examine DLLs associated with a process: Use the command `python3 vol.py -f /mnt/c/tools/volatility_2.6_win64_standalone/memdump.vmem dlllist --pid 5452` to view the DLLs associated with the TrustMe.exe process. This provides insight into the modules loaded by the process and its command line invocation.

Detect suspicious activities with malfind: Run the command `python3 vol.py -f /mnt/c/tools/volatility_2.6_win64_standalone/memdump.vmem windows.malfind.Malfind` to identify any suspicious activities within the processes. Look for memory areas with execute, read, and write permissions simultaneously, as this is uncommon and potentially indicative of malware.

Conclusion

By following these steps and analyzing the memory dump with Volatility, you can gain insights into network connections, processes, parent processes, DLLs, and potentially detect suspicious activities or malware present on the compromised system.
