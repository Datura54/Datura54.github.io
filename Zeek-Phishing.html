An alert triggered: 

"Phishing Attempt".
The case was assigned to you. Inspect the PCAP and retrieve the artefacts to confirm this alert is a true positive. 


First we will go to the pcap directory



And analyze “phishing.pcap"



Once it is done, we will check the table bellow to understand what might be the best way forward.



As this investigation was triggered by an alert, we should first see what “pe.log”



There is not enough information to work with, so we will keep moving forward by inspecting the content of “conn.log”



Out of the gate, “10.6.27.102” IP Address seems like a good starting point for our investigation. We will inspect “http.log” filtering for “10.6.27.102”

cat http.log | grep 10.6.27.102



Jackpot! Here we found that a user with the IP Address “10.6.27.102” downloaded an Document named “Invoice&MSO-Request.doc” and an executable “knr.exe” from the same domain address “smart-fax.com”.

Let's analyze this Domain with the help of VirusTotal



8 security vendors mark this domain as malicious, let's see what he says about the executable. To do that we need to either find it's MD5 or SHA256, or get the file and then just calculate it's hash.

We will inspect “file.log” content.

cat files.log | zeek-cut mime_type md5



Levereging VirusTotal, we will have the confirmation that this file is malicious.


So we found the IP Address of the malicious actor"10.6.27.102", the file and executable he injected in his target “Invoice&MSO-Request.doc”, “knr.exe” and by getting it's MD5 we confirmed the executable malicious profile by levereging VirusTotal.
