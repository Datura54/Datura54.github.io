First go to the bottom right side of your windows, left-click on the Network Icon then just left Click on your network

It will bring you to a page like this click on Change adapter options

You will have a page like this, now you will have to discover which one is the internet (NIC "Outside") and the internal (NIC "Internal")

For that you will have to see which one has a normal IP Address and which got an automated ip addressed by the VM, to do that you click on one of them, then Details, here you will analyze by the IP which is which, so on this one, you can see it's a strange IP 169.254.77.114 so this is our internal NAT

By Association the other one is the Internet NIC, which we can confirm here, to make everything easier change their names to the correspondent NIC (The strange IP=Internal NIC; Normal IP=Internet NIC)

Something like this

Now we will change the Internal NIC IP (Stange IP), do to that go to Properties -> (Double click) Internet Protocol Version 4 (TCP/IPv4) it should look like this

Now you will have to change to
IP Adress: 169.254.77.114
Subnet Mask: 255.255.255.0
Default Gateway: nothing (don't type anything, it's literally nothing)
Preferred DNS Server: 172.16.0.1
Alternate DNS Server: nothing
we will not use a Default Gateway because our server will serve as one.
we will use the Internal NIC as a DNS server for itself, to do that you could use just the IP address that you addressed a few seconds ago or use a generic IP Address (127.0.0.1) that refers to "myself"

Now we will change the name of our computer, to do that we will go Righ click on Windows Icon -> System -> Rename This PC, here

I changed to DC (Domain Controller), then just restart

#SetupDomain
So now, the NIC is all setup, next we will setup the Domain / AD DS, for that go to your Server Manager and click on Add roles and features

That should have opened a window like this, now you will just hit next except on the one talk address on the next print

So you once you are on Server Roles (There's an index on the left) You will select the second option Active Directory Domain Services

Add Features

Install, This may take some time, don't worry

After you hit close, you will see that you have a notification in the right-up corner. We installed the software but we haven't created the Domain yet, so to do it click on the notification

Promote this server to a domain controller

Add a new forest, i will use mydomain.com you can use whatever you want, hit next

Now put a password, we might never use it but choose something you will remember like bunny bee (it's not my password you don't need to try it, especially on my Gmail), hit next till install

After restarting, we will create our dedicated domain account, instead of using the built-in one. To do that, Windows menu -> Windows Administrative Tools -> Active Directory Users and Computer

Here is the domain that we created earlier, now we will create a dedicated one with the same attributes, so right-click on mydomain.com then follow the print. We could talk a lot about Organizational Units but for now, think of it as a folder.

You can uncheck the little box there, I will just not be annoying when later we delete it

It should look like that on the left side, now we will add a new User

You can name it whatever you like, I did like this. On the login name, I put the "a-" to signal this is an Admin

just create his password and check the boxes like so, we are checking the third box because we are in a Lab environment so we don't have to be bothered putting passwords

You will have something like this, so we now have an account but it's still not an admin one, we just named her admin. To make this user admin you have to go to his properties, (right click on him and then properties)

Member Of

Then click Add... it will open the Select Groups window, type on the big white box "Domain Admins" and then Check Names

If after you pressed Check Names your "Domain Admins" ended up looking like this, you did everything right, press OK and you will see that our user is now a Member of Users and Admins, press Apply,Ok and it's done you created a new user and added him to the Admin Members, so you created an Admin!


Now we will login on to our admin account, to do that sign out

And then login with another User, you will have to remember your User Logon name, if you forgot no problem go to your user -> properties -> Account and you will see it

Select that you want to login with another user, and just put the credentials you attributed to your admin

#SetupRAS/NAT
The purpose of this is so that when we create our Win10 Client he be in a private network but still will have access to the internet trough the DC, so what we will do is install NAT and RAS on to our Domain Controller to allow our Clients to do that

To do that we will go to Add roles and features

Hit next till a window looking like my print appears, here select Remote Acess then

Here select Routing, he will automatically select DiresctAccess and VPN (RAS)
Just hit next till install

Now go to Tools -> Routing and Remote Access**

Here select the Internet Interface and hit next (Here you can see that we made something wrong as on the internal NIC our IP address is 0.0.0.0, so I had to change the IP address again, if everything is ok for you great, you know it's ok if the ip you changed to is there)

You will end up with something like this

And its done, #SetupRAS/NAT now when we connect a client to our Domain Server he should be able to access the internet, assuming we setup DHCP, which is what we will do next we will setup a DHCP on our Domain Server

#SetupDCHP
Go to our Server manager, Add roles and features

Hit next till you find a window looking like the next print, Select DCHP Server and hit next till install

Now we will setup our scope(Range of IP that DHCP will give to the Server Clients) by going to Tools -> DHCP

New Scope...

On the next prints just read the intro from each window and you will know what they are

Here we will attribute the scope

Here is where you would add some Exclusions meaning you may want to reserve a special range to another server so you can play around with this kind of stuff

Here Is where you will say for how long an Ip address stays reserved for a device, meaning if you are at a Starbucks and get some coffee and then you go on with your life you won't need an IP The address is reserved for you for maybe more than 2 hours, but if you are in a Lab or Home environment then you might not switch devices that often making a refresh to the DCHP unnecessary

Here he is asking if we want to configure DHCP Options, which means that if we want to tell clients which Server to use for DNS or the Gateway, and we do want that

Click add before next

Just hit next till finish
